Audit item F2 asked to eliminate the ~37 "Token not allowed in a PDF string"
hyperref bookmark warnings in the supplement. Breakdown after Wave E4:

- **3 warnings** came from `supplement.tex` line 92 (the `$C_0$` section title).
  **Fixed by E4** with `\texorpdfstring{$C_0$}{C0}`. (main.tex: 0 such warnings.)
- **34 warnings remain**, and *all 34 originate in `appendix_body.tex`
  subsection titles* that contain math. E4 must not edit that file (you own it
  and are mid-edit), so they are handed off here.

The cause is unescaped math (`$…$`, `\ge`, `\bigstar`, `\Leftrightarrow`,
subscripts, primes) inside `\subsection{…}` titles. Fix each by wrapping the
math in `\texorpdfstring{<math>}{<plain text>}`. Exact suggested edits (line
numbers as of appendix_body.tex @ ~15:38 today — re-grep if you have shifted
them):

There is no clean preamble-only global fix: the `$` math-shift warning is
emitted per math group inside a bookmark string and can only be suppressed by
`\texorpdfstring` at the title itself. `\pdfstringdefDisableCommands` would only
silence the macro tokens (`\ge`, `\bigstar`, `\Leftrightarrow`), not the
dominant `$` warnings, so E4 did not apply a partial hack that would muddy the
count before your proper fix.
